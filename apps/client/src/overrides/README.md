# overrides/ — UI上書き（第3層・配布時）

団体/環境ごとに UI を上書きするためのディレクトリ。**ベースは未編集のまま**、ここに置いたものが優先される。

## 部分上書き（スロット注入）
`src/overrides/<slot名>.astro` を置くと、ベース画面の `<Slot name="<slot名>">…既定…</Slot>` がその内容に差し替わる。
例：`src/overrides/home-hero.astro` → ホームの hero スロットを独自内容に。

利用可能なスロット名は、ベース画面の `<Slot name="…">` を参照（例：`home-hero`）。

## 全面置換（ファイル上書き）
画面を丸ごと差し替えるときは、配布バンドルに `src/pages/<page>.astro` を同梱してベースファイルを上書きする。

## 配布
団体ごとの throwaway リポ／配布バンドルに本ディレクトリ（と必要なら上書き pages）を含める。
ベース（共通画面）は変更しないため、上流更新を取り込んでも上書きは保たれる。
